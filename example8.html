HTML Forms

HTML Forms

HTML forms are essential for collecting user input. Forms can contain various types of input fields, buttons, and other controls that allow users to enter data. They are typically wrapped in a <form> tag.

<form action="submit.php" method="post">
    <!-- Form elements go here -->
</form>

Example of a Basic Form

HTML Form Attributes

Forms have several attributes that control their behavior:

  • action: Specifies the URL where the form data will be sent for processing.
  • method: Defines the HTTP method to be used when sending form data. Common values are GET and POST.
  • enctype: Specifies how the form data should be encoded when submitting to the server.
  • target: Determines where to display the response after submitting the form.
  • name: Assigns a name to the form for identification in scripts.

HTML Form Elements

HTML forms can contain various elements, including:

  • <input>: The most common form element for user input.
  • <textarea>: For multi-line text input.
  • <select>: For dropdown menus.
  • <button>: For clickable buttons.
  • <fieldset>: Groups related elements within a form.
  • <label>: Associates a text label with a form element, improving accessibility.

HTML Input Types

The <input> element can have various types specified using the type attribute. Here are some common input types:

  • text: A single-line text input.
  • password: A password input that hides the entered characters.
  • email: An input for email addresses with validation for format.
  • number: An input for numeric values with up/down arrows for selection.
  • checkbox: A checkbox for boolean (true/false) input.
  • radio: A radio button for selecting one option from a group.
  • date: An input for selecting a date.
  • file: An input for uploading files.
  • submit: A button to submit the form.

HTML Input Attributes

Input elements can have various attributes to enhance their functionality:

  • name: The name of the input, used to identify the field when the form is submitted.
  • value: The initial value of the input field.
  • placeholder: A short hint that describes the expected value of the input field.
  • required: A boolean attribute that indicates the field must be filled out before submitting the form.
  • readonly: Prevents the user from modifying the input field.
  • disabled: Disables the input field, preventing user interaction.
  • min and max: Specify the minimum and maximum values for number and date input types.

Input Form Attributes

In addition to the attributes of individual inputs, forms themselves can have attributes that enhance functionality:

  • autocomplete: Controls whether the browser should enable autocomplete for input fields.
  • novalidate: Disables HTML5 validation when the form is submitted.
  • accept-charset: Specifies the character encodings the server can handle for the form submission.

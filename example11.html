HTML Media Tutorial

HTML Video

HTML5 introduced the <video> element for embedding videos directly into web pages without plugins.

Basic Video Example:

<video width="320" height="240" controls>
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.ogg" type="video/ogg">
    Your browser does not support the video tag.
</video>

Live Example:

Video Attributes:

Attribute Description
controls Adds video controls (play/pause/volume)
autoplay Video will start playing automatically
loop Video will start over when finished

HTML Audio

The <audio> element allows you to embed sound content in your web pages.

Basic Audio Example:

<audio controls>
    <source src="music.mp3" type="audio/mpeg">
    <source src="music.ogg" type="audio/ogg">
    Your browser does not support the audio element.
</audio>

HTML Plug-ins

While modern HTML5 features have largely replaced plugins, some legacy content might still require them.

Object Element Example:

<object data="file.pdf" type="application/pdf" width="100%" height="500px">
    <p>Unable to display PDF file. <a href="file.pdf">Download</a> instead.</p>
</object>

HTML YouTube

YouTube videos can be easily embedded using an iframe.

YouTube Embed Example:

<iframe 
    width="560" 
    height="315" 
    src="https://www.youtube.com/embed/z6zFKgJ5TuI?si=bKNpyY1Cs5upQrlt" 
    title="YouTube video player" 
    frameborder="0" 
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" 
    referrerpolicy="strict-origin-when-cross-origin" 
    allowfullscreen>
</iframe>

Live YouTube Example:

Key YouTube Embed Attributes:

Attribute Description
src The URL of the YouTube video (embed format)
title Descriptive title for accessibility
frameborder Specifies whether to display a border around the iframe
allow Specifies which features are allowed in the iframe
referrerpolicy Specifies which referrer information to send
allowfullscreen Enables fullscreen mode option

Best Practices:

  • Always include width and height attributes
  • Use responsive design for better mobile experience
  • Consider loading time and page performance
  • Provide fallback content for accessibility
